Grundsätzliche Funktion
- Die zu entwickelnde Software, stellt eine benutzerfreundlichere Schnittstelle zur Benutzung des Kommandozeilentools Osmosis bereit. Dabei handelt es sich  um ein Werkzeug zur Pipline gestützten Verarbeitung von OpenStreetMap Daten, dabei können diese aus einer XML Datei oder einer Datenbank gelesen werden und anschließend durch verschiedenste Tasks gefiltert, verschmolzen, in Layer aufgetrennt und verändert (z.B. durch das Einfließen lassen von Änderungen) werden. Hierzu soll das Pipeline Modell welches von Osmosis verwendet wird auf ein leicht benutzbares GUI Modell übertragen werden. 
Ist Zustand:
Wer nutzt Osmosis?
- Das OpenStreetMap Projekt, aber eigentlich alle die es nutzten wollen
Welche Vorkenntnisse können Vorrausgesetzt werden:
- Nutzer haben gewisse Osmosis Vorkenntnisse – wir können ein Verständnis des Pipline Models annehmen
Funktionalität:
- Volle Funktionalität von Osmosis sollte zugänglich sein allerdings sollte die Komplexität versteckt werden, soweit möglich.
- Das Programm muss mindestens eine heraus kopierbare Kommandozeile für Osmosis liefern, sowie die Möglichkeit sie als ausführbare Scripte (bat,sh) zu Exportieren
diese Skripte müssen auch importiert werden können dies fúhrt zu folgenden Szenarien:
   1. import/laden:vorhandene Kommandozeile → Pipelinemodel (unser Programm)
   2. export/speichern: Pipelinemodel → Kommandozeile (sh, bat)
- Boundingboxes sollen auf einer Karte grafisch ausgewählt werden können, API hierfúr kommt vom Kunden als Beistellung.
- Die Skizze ist ein Schnellentwurf und nur eine grobe vorstellung.
- Die GUI muss mindestens Deutsch und Englisch unterstützen, das dahinter liegende Lokalisierungsmodell soll erweiterbar bleiben
- Barrierefreiheit wäre nett, aber ist nicht  zwingend gefordert
- Die Software muss auf allen Maschinen mit Java SE 1.6 lauffähig sein sie ist mit Java zu schreiben, externe Bibliotheken müssen genehmigt werden.
- Die Software muss ab einer Auflösung von 1024x768 benutzbar seinin)
- Es bestehen keine besonderen gesetzlichen Normen oder Richtlinien für die GUI, jedoch sollte sie einfach zu bedienen sein und das generelle Look&Feel anderer Java Swing Anwendungen beibehalten
- Die Software muss Piplines bis mindestens 100 Tasks verwalten können
- Osmosis 0.37 ist Referenz (die Software sollte allerdings leicht erweiterbar und für neue Versionen anpassbar sein)
- Die Software soll gut Anpassbar sein
- Zugangsdaten von zu verwendenden Datenbanken (Verwendung erfolgt durch Osmosis) sollten speicherbar sein.
- Die erzeugten Scripte werden vom Kunden wie übliche Dateien im Dateisystem verwaltet, es erfolgt keine Verwaltung durch unsere Software
- Selbstverständlich soll die Software keine Daten des Kunden oder die Sicherheit seines Systems gefährden

Einige Vormerkungen für das Begrifflexikon:
- Modul/Task
- Pipeline: Daten aus einer Datenquelle (z.b. eine XML Datei oder Datenbank) werden von Tasks wie an einem Fließband verarbeitet bis die bearbeiteten Daten wieder in eine Datei oder eine Datenbank geschrieben werden
-Task: --boundingbox + parameter (Verarbeitungsschritte)
- Online Handbuch: ein aus der Applikation heraus aufrufbares Handbuch
